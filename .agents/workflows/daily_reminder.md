## Project Reminder: Change Tracking

Every time you start a session in the GhostPost project, you MUST:

1.  Load this workflow and review the `CHANGELOG_SCRATCHPAD.md` file in the project root.
2.  Append a summary of ALL changes made during each task to `CHANGELOG_SCRATCHPAD.md`.
3.  Each entry MUST include:
    - Date and Time Stamp
    - Task Name/Description
    - Summary of Changes
    - Final Timestamp

### Example Entry:
```markdown
## 2026-03-23 16:45:00

- **Task:** [Task Name]
- **Summary:** [Detailed summary of changes]
- **Timestamp:** 2026-03-23T16:45:00-04:00
```

> [!IMPORTANT]
> This scratchpad is critical for maintaining consistency and history across sessions. Never skip updating it.
